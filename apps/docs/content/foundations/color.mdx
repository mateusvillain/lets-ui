# Color

The Let's UI color system is built on a two-layer architecture that separates raw palette values from their intended meaning. This structure makes theming predictable, dark mode automatic, and brand customization possible without touching component code.

## Color system

Foram projetadas ao todo 6 paletas no sistema de cores, seguindo o mesmo princípio de construção, contando com 8 níveis de tonalidade, utilizando o modelo LCH (Lightness, Chroma, Hue) na construção das escalas para ter maior fidelidade com a maneira como os seres humanos percebem as diferenças de cor.

## Properties

### Background

Os tokens de background definem as superfícies visuais que estruturam a interface. Eles são responsáveis por criar separação hierárquica, estabelecer contraste entre camadas e apoiar a legibilidade dos elementos de conteúdo e interação. Devem ser aplicados considerando contexto, profundidade visual e consistência entre diferentes estados e temas da interface.

### Border

Os tokens de border definem limites visuais, separações estruturais e contornos de componentes. Seu papel principal é reforçar organização, delimitar áreas funcionais e auxiliar na percepção de agrupamento e hierarquia da interface. O uso deve priorizar clareza visual sem tornar a experiência excessivamente carregada.

### Text

Os tokens de texto definem os níveis de legibilidade, hierarquia e comunicação visual da interface. Sua aplicação deve considerar contraste, densidade de informação, acessibilidade e contexto de uso, garantindo leitura consistente em diferentes superfícies e estados visuais.
